# 🤖 AI Telegram-бот «Центр Красок #1»

Telegram-бот с AI-ассистентом, который отвечает на вопросы о компании [Центр Красок #1](https://centr-krasok.kz/) на основе собранной базы знаний.

**Бесплатный AI** — используется Groq API с моделью `llama-3.3-70b-versatile`.

---

## 📁 Структура проекта

```
centr_krasok_bot/
├── bot.py                  # Основной файл бота
├── company_knowledge.py    # База знаний о компании
├── requirements.txt        # Python-зависимости
├── Dockerfile              # Docker-образ для деплоя
├── .env.example            # Шаблон переменных окружения
└── README.md               # Документация
```

## 🚀 Быстрый старт

### 1. Получить токены

**Telegram Bot Token:**
1. Откройте Telegram, найдите [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot`
3. Задайте имя и username бота
4. Скопируйте токен

**Groq API Key:**
1. Зайдите на [console.groq.com](https://console.groq.com)
2. Зарегистрируйтесь (можно через Google/GitHub)
3. Перейдите в **API Keys** → **Create API Key**
4. Скопируйте ключ (начинается с `gsk_`)

---

### 2. Запуск локально

```bash
# 1. Клонируйте / распакуйте проект
cd centr_krasok_bot

# 2. Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate       # Linux/Mac
venv\Scripts\activate.bat      # Windows

# 3. Установите зависимости
pip install -r requirements.txt

# 4. Задайте переменные окружения
export TELEGRAM_TOKEN="ваш_токен"
export GROQ_API_KEY="ваш_groq_ключ"
# Windows: set TELEGRAM_TOKEN=...
# Windows: set GROQ_API_KEY=...

# 5. Запустите бота
python bot.py

---

### 3. Запуск через Docker

```bash
# Сборка образа
docker build -t centr-krasok-bot .

# Запуск контейнера
docker run -d \
  -e TELEGRAM_TOKEN="ваш_токен" \
  -e ANTHROPIC_API_KEY="ваш_ключ" \
  --name ck-bot \
  --restart unless-stopped \
  centr-krasok-bot
```

---

### 4. Деплой на Railway (бесплатно)

1. Зайдите на [railway.app](https://railway.app)
2. Создайте новый проект → Deploy from GitHub
3. Загрузите код в свой GitHub-репозиторий
4. В настройках Railway добавьте переменные:
   - `TELEGRAM_TOKEN`
   - `ANTHROPIC_API_KEY`
5. Railway автоматически запустит бота по Dockerfile

## 🧠 Как работает бот

### Архитектура

```
Пользователь (Telegram)
        ↓ сообщение
  python-telegram-bot
        ↓
  Сборка контекста диалога (история)
        ↓
  Anthropic Claude API
  (системный промпт + база знаний + история)
        ↓
  Ответ на основе базы знаний
        ↓
  Пользователь получает ответ
```

### Контекст диалога
Бот хранит историю последних **10 пар** (вопрос + ответ) для каждого пользователя. Это позволяет поддерживать связный диалог — например, если пользователь сначала спросил про краску, а потом уточняет «а какой бренд лучше?», бот понимает контекст.

### Защита от галлюцинаций
Системный промпт явно ограничивает модель:
- Отвечать **только на основе базы знаний**
- При отсутствии данных — направлять к менеджеру
- Не отвечать на вопросы, не связанные с компанией

## 📊 Архитектурные решения

### Почему не RAG (векторная база)?
Для MVP с одной компанией весь контекст умещается в одном промпте (~3000 токенов). RAG оправдан при сотнях документов. Данное решение проще и надёжнее.

### Почему Claude Sonnet, а не Haiku?
Sonnet обеспечивает лучшее следование инструкциям (меньше «галлюцинаций») при приемлемой стоимости.

### Хранение истории
В памяти процесса — достаточно для MVP. При масштабировании на несколько инстансов нужно вынести в Redis.
